# Gorilla Generator: ASP.NET Validation & JSON Converter Improvements

## Problem

Discriminated unions generated by Zooper.Gorilla extend `OneOfBase<T0, T1, ...>` from the OneOf library. This causes two issues:

### 1. ASP.NET Model Validation Crash

When a discriminated union is used as a property in an ASP.NET controller request DTO, the framework's `ValidationVisitor` walks the full object graph during model binding. It accesses all public properties on `OneOfBase` — including `AsT0`, `AsT1`, etc. Accessing the inactive variant throws:

```
System.InvalidOperationException: Cannot return as T1 as result is T0
   at OneOf.OneOfBase`2.get_AsT1()
   at Microsoft.AspNetCore.Mvc.ModelBinding.Validation.ValidationVisitor.VisitChildren(...)
```

This affects every endpoint that accepts a discriminated union in its request body. Currently, consumer projects must work around this by registering a custom `IValidationMetadataProvider` that suppresses child validation for `IOneOf` types.

### 2. Manual JSON Converters

Every discriminated union requires a hand-written `JsonConverter` in the consuming project (e.g. `CreateProfileDtoConverter`, `ExternalReferenceDtoConverter`). These converters all follow the same pattern: discriminate on a `"type"` field and map to the correct factory method.

## Proposed Changes

### Fix 1: Generate `[ValidateNever]` Attribute

For each `[DiscriminatedUnion]`, the generator should emit a `[ValidateNever]` attribute (from `Microsoft.AspNetCore.Mvc.ModelBinding.Validation`) on the generated partial class.

**Before** (current output):
```csharp
public sealed partial class CreateProfileDto : OneOfBase<CreateProfileDto.PersonData, CreateProfileDto.CompanyData>
{
    // ...
}
```

**After** (desired output):
```csharp
[System.ComponentModel.DataAnnotations.Schema.NotMapped] // optional, also helps with EF
[Microsoft.AspNetCore.Mvc.ModelBinding.Validation.ValidateNever]
public sealed partial class CreateProfileDto : OneOfBase<CreateProfileDto.PersonData, CreateProfileDto.CompanyData>
{
    // ...
}
```

This prevents ASP.NET from walking into `OneOfBase` properties during validation, fixing the crash at the source.

**Consideration:** The `Microsoft.AspNetCore.Mvc.ModelBinding.Validation` namespace requires a dependency on `Microsoft.AspNetCore.Mvc.Core`. To avoid forcing this dependency on non-web consumers, this could be:
- An opt-in generator flag (e.g. `[DiscriminatedUnion(SuppressValidation = true)]`)
- A separate companion package (e.g. `Zooper.Gorilla.AspNetCore`)
- Emitted only when the consuming project already references ASP.NET MVC

### Fix 2: Auto-Generate JSON Converters

The generator should produce a `JsonConverter<T>` for each discriminated union, using the variant name as the discriminator value in a `"type"` field.

**Given this input:**
```csharp
[DiscriminatedUnion]
public sealed partial class CreateProfileDto
{
    [Variant]
    public static partial CreateProfileDto Person(string firstName, string lastName);

    [Variant]
    public static partial CreateProfileDto Company(CompanyName companyName);
}
```

**The generator should produce a converter that handles this JSON format:**
```json
{ "type": "Person", "firstName": "John", "lastName": "Doe" }
{ "type": "Company", "companyName": "Acme Corp" }
```

The converter should also be automatically registered on the type via a `[JsonConverter(typeof(...))]` attribute, so consumer projects don't need to manually register it.

**Considerations:**
- Support both `System.Text.Json` and `Newtonsoft.Json` (possibly via separate packages or a generator flag)
- The discriminator field name (`"type"`) could be configurable via an attribute parameter
- Variant property names in JSON should follow camelCase by default
